### （1）题目意思与生僻概念解释

**题目意思：**
给定一个数字 `n`，代表手里有 `n` 个左括号 `(` 和 `n` 个右括号 `)`。要求把这 $2n$ 个括号全部拼凑起来，形成完整的字符串，并且要求括号的开闭必须是“合法的”（即每一个右括号都必须有它前面对应的左括号相匹配）。输出所有能够拼成的合法结果。

**核心生僻概念解释：**
* **合法括号组合（Valid Parentheses）：** 具备两个硬性条件的字符串：
    1.  最终左括号和右括号的总数量必须相等。
    2.  在任意时刻，从左往右数括号时，**右括号的数量绝对不能超过左括号的数量**。一旦超过，必定是非法闭合。
* **深度优先搜索（DFS）与回溯（Backtracking）：** 一种“走迷宫”式的算法思想。在每一步都尝试所有可能的选择（选左括号或右括号），顺着一条路一直走，如果发现走到死胡同（不符合合法括号的规则）或者走到了终点（拿到完整字符串），就**退回上一个岔路口**，换另一种选择继续走，直到走遍所有可能的合法路径。

### （2）代码解题思路解析

修正后的回溯算法核心思路是**“带条件限制的分岔探索”**。

在每一次决定向字符串里追加哪个括号时，严格遵循以下三条规则：
1.  **左括号随时可以加（前提是没用完）：** 只要当前消耗的左括号数量小于总数 `n`，就可以放心地追加左括号。
2.  **右括号必须滞后加：** 只有当目前字符串里的**右括号数量严格小于左括号数量**时，才能追加右括号。这是避免生成非法括号组合（如 `())`）的核心“剪枝”操作。
3.  **终点结算：** 当拼接的字符串长度刚好等于 `n * 2` 时，说明一组完整的括号已经拼装完毕，将其记录到最终的结果列表中，并停止该分支的继续探索。

### （3）带有注释的代码框

```python
class Solution:
    def generateParenthesis(self, n: int) -> List[str]:
        # 排除边界情况，若对数小于等于 0 则返回空列表
        if n <= 0: return []
        # 初始化全局结果列表，用于收集最终所有的合法括号字符串
        res = []
        # 定义带有三个状态参数的回溯辅助函数
        # paths: 当前正在拼接的字符串
        # left: 当前已经使用的左括号数量
        # right: 当前已经使用的右括号数量
        def dfs(paths: str, left: int, right: int):
            # 终止条件：当字符串长度达到 2n 时，说明拼装完成
            if len(paths) == n * 2:
                res.append(paths)
                return
            # 探索分支 1：只要左括号还没用完，就可以追加左括号
            if left < n:
                dfs(paths + '(', left + 1, right)
            # 探索分支 2：只要右括号比左括号少，就可以追加右括号闭合
            if right < left:
                dfs(paths + ')', left, right + 1)
        # 触发首次搜索：初始字符串为空，左右括号使用量均为 0
        dfs('', 0, 0)
        return res
```

### （4）每一行代码详细中文解释

1.  `class Solution:`
    * **解释：** 定义算法平台标准的题解类。
2.  `def generateParenthesis(self, n: int) -> List[str]:`
    * **解释：** 定义主方法，接收整数 `n` 代表括号对数，返回一个由字符串构成的列表。
3.  `if n <= 0: return []`
    * **解释：** 边界防御代码。如果要求的对数小于等于 0，不可能生成括号，直接返回空列表。
4.  `res = []`
    * **解释：** 初始化一个空列表 `res`，用于存放所有在探索终点收集到的合法括号组合。
5.  `def dfs(paths: str, left: int, right: int):`
    * **解释：** 在主方法内部定义辅助深度优先搜索函数。携带三个极其关键的状态：当前累积的字符串 `paths`，已经消耗的左括号数 `left`，以及消耗的右括号数 `right`。
6.  `if len(paths) == n * 2:`
    * **解释：** 检查当前的字符串长度是否达到了应有的最终长度（`n` 对括号对应 $2 \times n$ 个字符）。
7.  `res.append(paths)`
    * **解释：** 如果长度达标，说明这是一条走到终点的合法路径。将当前成型的字符串添加到全局结果 `res` 中。
8.  `return`
    * **解释：** 终点结算完毕，直接结束当前递归分支，触发回溯动作，退回上一层去探索其他可能。
9.  `if left < n:`
    * **解释：** 检查已经消耗的左括号数量是否小于规定的总对数 `n`。这是能否添加左括号的前提。
10. `dfs(paths + '(', left + 1, right)`
    * **解释：** 如果允许加左括号，则在字符串末尾拼接 `(`，同时将左括号消耗量 `left` 加 1，右括号数量不变，向更深一层递归。
11. `if right < left:`
    * **解释：** 检查已经消耗的右括号数量是否严格小于已经消耗的左括号数量。这是防止括号序列失效的核心约束。
12. `dfs(paths + ')', left, right + 1)`
    * **解释：** 如果允许加右括号，则在末尾拼接 `)`，此时左括号消耗量不变，右括号消耗量 `right` 加 1，向更深一层递归。
13. `dfs('', 0, 0)`
    * **解释：** 辅助函数定义完毕，正式拉开探索的序幕。从空字符串开始，左括号用量为 0，右括号用量为 0。
14. `return res`
    * **解释：** 当最初的那次 `dfs` 调用彻底结束，搜索树的所有合法分支都被遍历完毕，返回装满答案的结果集。

### （5）具体数值算例与变量变化过程

**设定算例：** 输入 `n = 2`
最终需生成长度为 4 的组合。以下呈现回溯探索树从左到右深度优先展开的核心步骤。

| :--- | :--- | :--- | :--- | :--- | :--- |
| **初始启动** | `""` | `0` | `0` | 开启回溯大门。 | `dfs('', 0, 0)` |
| ├── **第1层深入** | `"("` | `1` | `0` | `left(0) < 2` 成立，追加左括号。 | `if left < n: dfs(paths + '(', ...)` |
| │   ├── **第2层深入** | `"(( "` | `2` | `0` | `left(1) < 2` 成立，继续追加左括号。 | `if left < n: dfs(paths + '(', ...)` |
| │   │   └── **第3层深入** | `"(()"` | `2` | `1` | `left(2)<2` 不成立。但 `right(0) < left(2)` 成立，追加右括号。 | `if right < left: dfs(paths + ')', ...)` |
| │   │       └── **第4层深入** | `"(())"` | `2` | `2` | 同上，`right(1) < left(2)` 成立，追加右括号。 | `if right < left: dfs(paths + ')', ...)` |
| │   │           └── **触底结算**| `"(())"` | `2` | `2` | 长度达到 $2 \times 2 = 4$，完成合法组合，存入结果集并 `return`。 | `if len(paths) == n * 2: res.append(...)` |
| │   └── **回退后并进**| `"()"` | `1` | `1` | 回退至第1层 `paths="("`。执行下方的 `right(0) < left(1)` 成立，追加右括号。 | `if right < left: dfs(paths + ')', ...)` |
| │       └── **重新深入** | `"()("` | `2` | `1` | 基于 `"()"` 继续。`left(1) < 2` 成立，追加左括号。 | `if left < n: dfs(paths + '(', ...)` |
| │           └── **继续深入** | `"()()"` | `2` | `2` | `left` 已满，`right(1) < left(2)` 成立，追加右括号。 | `if right < left: dfs(paths + ')', ...)` |
| │               └── **触底结算**| `"()()"` | `2` | `2` | 长度达到 $2 \times 2 = 4$，完成第二组合法组合，存入结果集。 | `if len(paths) == n * 2: res.append(...)` |
| └── **回退至起点** | `""` | `0` | `0` | 在起点尝试右分支：`right(0) < left(0)` 不成立，拒绝非法起步，探索完全结束。 | `if right < left:` 阻断非法分支 |

最终结果集输出为：`["(())", "()()"]`。
